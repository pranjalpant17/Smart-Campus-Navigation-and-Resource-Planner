Adjacency List — O(V+E) Space
Adjacency List: Each building stores only its direct neighbours as a linked list. Space-efficient for sparse graphs. Used by BFS, DFS, and Dijkstra. Adjacency Matrix: 2D array — O(1) edge weight lookup, used during path reconstruction.

BFS — Minimum Hops Path
BFS uses a Queue (FIFO). It explores all nodes at distance 1, then 2, then 3… so the first time it reaches the destination, that is guaranteed to be via the fewest edges (hops) — not necessarily the shortest distance. Time: O(V + E)
Result Select buildings and click Run BFS.
DFS — All Simple Paths (max 20)
DFS dives deep along one branch, then backtracks (unmarks visited) to explore alternatives. This finds ALL simple paths. Uses the call stack recursively. Time: O(V + E) per path — total paths can be exponential.
Result Select buildings and click Find All Paths.
Dijkstra — Shortest Weighted Path
Dijkstra uses a Min-Heap Priority Queue. Always picks the unvisited node with the smallest known distance, then relaxes its neighbours. Finds the minimum total distance in metres — not hop count. Time: O((V+E) log V)
Result Select buildings and click Run.
